## Зафиксированные инварианты

1. `RunEngine` отвечает только за драфт; `TournamentEngine` — отдельная чистая orchestration-модель.
2. Турнир не вызывает внешние API и использует загруженный static-first snapshot.
3. Прогноз и фактический турнир строятся одним engine, а не двумя несогласованными формулами.
4. Resume хранит только число открытых этапов и воспроизводит результат из seed.
5. Ни одну стадию нельзя перескочить; после `complete` дальнейший `advance` запрещён.
