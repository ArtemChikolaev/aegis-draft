## Что вообще есть у 322-0 (замер, не впечатление)

Вся их система движения — **три класса**:

```
.card-in    cardIn .38s cubic-bezier(.2,.8,.2,1)   translateY(10px) scale(.95) → none
.fade-in    fadeIn .28s ease                        translateY(6px) → none
.card-flip  flipIn .5s cubic-bezier(.2,.8,.2,1)     rotateY(85deg) → 0
transitions: Tailwind-дефолты — .15s (цвета), ease cubic-bezier(.4,0,.2,1)
```

Остальные 28 `@keyframes` (`orbit*`, `wisp*`, `wave*`, `beam`, `disk`, `haze`, `lens`, `sprite`) — анимированный арт лоадера, к игровому циклу отношения не имеют.

**Ключевая находка — раздача пака.** В живом DOM 10 карт (`.card-flip`) с шагом задержки ровно 0.12s:

```
игроки: 0s, 0.12s, 0.24s, 0.36s, 0.48s
герои:  0.6s, 0.72s, 0.84s, 0.96s, 1.08s     каждая — flipIn .5s
```

Полный расклад — 1.58s. Повторяется **на каждый новый пак** (проверено: пик → `Pack: OG · manila2016` сменился на `Pack: Virtus.pro · ti2019`, все 10 карт перевернулись заново). Это и есть ритм их драфта.

## Приоритеты

**P1 — раздача пака (`missing`).**
Единственный настоящий пробел. У них 1.58s тактильной обратной связи на каждый пак; у нас пак возникает мгновенно, без единого анимированного узла. Это не косметика: раздача задаёт ритм и отделяет «новый пак» от «тот же пак» — сейчас после пика экран меняется без сигнала, и глазу не за что зацепиться.

**P2 — count-up для TEAM OVR.**
У них скачок, у нас скачок — паритет. Но число в центре пентагона это главный фидбек драфта, и плавный набег читался бы лучше **чем у референса**. Улучшение, а не догоняние.

**P2 — форма пака (карты vs список).**
Осознанное расхождение, не дефект. Их карта 3:4 приглашает выбрать; наш список информативнее (влезло название события — у них только `Pack: OG · manila2016`). Менять — продуктовое решение, не для аудита.

P0 нет.
