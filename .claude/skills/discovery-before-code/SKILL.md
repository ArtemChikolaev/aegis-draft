# Discovery before code — сначала найди готовое

За 5–15 минут до первой строки кода ответь: **«что уже есть в проекте, и чего я НЕ буду писать»**. Дубликаты и рассинхрон контракта данных рождаются из пропуска этого шага.

## Когда применять
- Новый Go-пакет/функция в `pipeline/internal/*` — сначала проверь соседние стадии (`fetch/normalize/aggregate/rate/emit`), нет ли уже клиента/утилиты.
- Новый TS-модуль в `web/src/*` — логика счёта (`game/score.ts`), генерация паков (`game/packs.ts`), назначение героев (`game/assign.ts`), стор (`state/`) — почти всё уже размечено в `web/README.md`.
- **Новая структура данных или поле** — сначала посмотри, нет ли её в `schema/*.schema.json`. Контракт данных — единый источник (см. [[data-contract]]).
- **Рейтинг/формула** — не изобретай: модель зафиксирована в PRD §5 и [[scoring-model]].
- **Внешний источник** (OpenDota/Liquipedia) — клиент и правила уже описаны в [[external-data-etl]].

## Инструкции
1. **Пойми задачу (≤3 мин).** Сформулируй своими словами, что должно работать. Выпиши golden path + edge cases. Прочитай все артефакты пользователя.
2. **Прочитай карту (≤2 мин).** `README.md` (структура), `docs/PRD.md` (механика/решения), `schema/README.md` (контракт), `pipeline/README.md`, `web/README.md`.
3. **Грепни по семантике, не по имени (≤5 мин).**
   - Go: `grep -rn "func " pipeline/internal | grep -i <тема>`; типы — `pipeline/internal/model`.
   - TS: `grep -rn "export " web/src | grep -i <тема>`.
   - Схема: `ls schema/ && grep -rn "<поле>" schema/`.
4. **Сформулируй reuse-резюме одной строкой:**
   > «Я НЕ создаю X, Y — есть A, B. Создаю только Q, аналога нет.»
5. **Стоп-сигналы — спроси пользователя** (см. [[plan-first-communication]]): два почти-подходящих кандидата; готовое почти подходит, но нужно расширение; интуиция «должно быть», но не нашёл.

## Что НЕ делать
- Не «грепнул точное имя из задачи → не нашёл → пишу своё». Греп по категориям.
- Не добавляй поле в data JSON, не обновив `schema/` (рассинхрон контракта — см. [[data-contract]]).
- Не переизобретай формулу рейтинга/генерацию паков — они в PRD §5 / [[scoring-model]].

## Связано
- Дальше → [[plan-first-communication]] (если задача нетривиальна).
- Правка модели данных → [[data-contract]]. Рейтинги → [[scoring-model]]. Внешние данные → [[external-data-etl]].
- Перед «готово» → [[self-review-checklist]].
