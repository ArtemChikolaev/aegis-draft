# Self-review — пройди до того, как сказать «готово»

Прогон мысленно каждого изменённого файла. Один «да, проблема есть» → иди и почини.

## Когда применять
- Перед сообщением «готово, посмотри», перед коммитом/PR.
- После того как локально заработало (тут чаще всего пропускают ревью).

## Алгоритм
1. Автоскан: `bash .claude/skills/self-review-checklist/tools/antipatterns_grep.sh`.
2. Пройди чек-лист ниже мысленно по каждому файлу.
3. Diff-обзор: нет ли несвязанных правок (чужое форматирование, мини-рефакторы вне scope) — убери.
4. Статика/сборка:
   - Go: `gofmt -l pipeline` (пусто), `go vet ./...`, `go build ./...`.
   - TS: `npm run typecheck` / `tsc --noEmit`, линтер чист.
5. Данные: менял `schema/` или data JSON → провалидируй (см. [[data-contract]]) и синхронизируй оба конца.
6. Рейтинги: менял формулу → бампнул `ratingModelVersion` (см. [[scoring-model]]).

## Категории проверки
### Reuse / архитектура
- Написал новое, хотя есть готовое (см. [[discovery-before-code]])? → переиспользуй.
- Границы: Go `internal/*` стадии не лезут друг в друга через глобалы; TS `game/` (логика) не зависит от `ui/`; доступ к данным — через `DataSource`, не прямой fetch по компонентам.
- Секреты (ключ OpenDota, контакт UA) не в коде, а в env.

### UI / design-system (если трогал `web/src/ui|features|app|design`, см. [[frontend-architecture]])
- Новый UI-элемент собран из примитивов `ui/` (или добавлен variant), а не bespoke `<div>` со стилями.
- **Ноль хардкод-цветов** (`#hex`, `rgba`) в компонентах/tsx — только токены `design/tokens.css`; нового `html[data-theme=…]`-override нет (значит цвет токенизирован).
- Весь видимый текст — ключ `i18n/core.ts` (RU + EN синхронно), не хардкод-строка. Примитивы `ui/` презентационные (i18n снаружи).
- Экран — в `features/`, примитив — в `ui/` (не наоборот). Проверил light + dark вживую.

### Контракт данных
- Тронул формат данных, не тронув `schema/` → рассинхрон. `steamId64` допустим только как raw-вход в `pipeline/internal/normalize`; после normalize и во всех output — единый `accountId`.
- Захардкодил формат в TS вместо генерации из схемы.

### Дубликаты / мёртвый код
- Две функции отличаются на 1 строку → параметризуй.
- Скопированный кусок между `pipeline`/`web` → общий модуль/справочник.
- Закомментированный код, `fmt.Println`/`console.log`-отладка, TODO без owner → убрать.
- Неиспользуемые импорты/переменные/экспорты.

### Скоринг
- Сырой winrate при малых выборках (нет сглаживания/порога).
- Жадное назначение героев вместо matching.
- Роли поделены на 4/5 (запрещено, PRD §5.1).

### Edge cases
- Пустой пул паков по формату; игрок без hero-stats; пара без squadSynergy; окно Peak без `N_min` игр; бесконечные рерроллы (Easy) не ломают состояние.
